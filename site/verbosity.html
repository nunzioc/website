Verbosity is Non-Linear

TEMP: this is just a draft for now.

I think most programmers can agree that code should focus on being easy to read rather than easy to write. When getting stuff done at the command line it's fun to cd and ls our way around but no one wants maintained software filled with tiny two letter verbs. But "easy to read" means very different things to different people. You can especially see how different progamming language communities come to wildy differing standards of what good code looks like.
